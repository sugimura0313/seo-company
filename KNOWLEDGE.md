# SEO・WordPress 知識ベース

## SEO基本戦略

### キーワード選定の原則
- **ロングテール優先**: 月間検索数100〜1,000のKWで勝負（競合が弱い）
- **検索意図の4分類**: 情報収集型 / 比較検討型 / 購買型 / ナビゲーション型
- **KGR（Keyword Golden Ratio）**: allintitle検索数 ÷ 月間検索数 < 0.25 が狙い目
- **LSIキーワード**: 共起語・関連語を本文に自然に含める

### 記事構成の鉄則
- タイトルに数字・年月・「方法」「コツ」「完全ガイド」を入れる
- リード文で「誰向け」「何がわかる」「読む価値」を30秒で伝える
- H2は検索意図に直接答える（ユーザーの疑問＝見出し）
- まとめ+CTAは必須（次の行動を明示）

### 内部リンク設計
- **サイロ構造**: カテゴリページ（柱記事）→ 個別記事（クラスター）
- 柱記事から全クラスターへリンク、クラスターから柱記事へリンク
- アンカーテキストはKWを含む自然な文言

## WordPress知見

### 推奨プラグイン（SEO特化）
| プラグイン | 用途 |
|------------|------|
| Rank Math / Yoast SEO | メタ設定・サイトマップ・構造化データ |
| WP Rocket / LiteSpeed Cache | ページ速度最適化 |
| Smush / ShortPixel | 画像圧縮 |
| Schema Pro | 構造化データ強化 |
| Broken Link Checker | 内部リンク管理 |

### Core Web Vitals 改善チェックリスト
- [ ] LCP < 2.5秒（画像最適化・CDN）
- [ ] FID < 100ms（JS遅延読み込み）
- [ ] CLS < 0.1（画像にwidth/height指定）
- [ ] TTFB < 800ms（サーバー・キャッシュ）

### 構造化データ（必須）
- Article / BlogPosting
- BreadcrumbList
- FAQPage（Q&Aセクションがある記事）
- LocalBusiness（地域SEO案件）

## 競合分析の手順
1. 狙うKWでGoogleトップ10を確認
2. 上位3記事の文字数・見出し構成・内部リンク数を調査
3. 自社記事は「上位記事より詳しい・わかりやすい・新しい」を目指す
4. 上位記事にない一次情報（体験談・事例・独自調査）を追加
